# Образцы записей для STT-сводки

## 1. Положи записи
2-5 коротких **.wav** (свои тестовые, не реальных клиентов — ради приватности).
Поддержка: `.wav .mp3 .m4a .ogg .webm`.

## 2. (Необязательно, но именно это включает авто-сверку) Напиши «ключ ответов»
Скопируй `expected.example.json` → `expected.json` и впиши что РЕАЛЬНО было в каждой
записи: поздоровался ли, сколько клиентов, была ли грубость и т.д. Указывай только то,
что знаешь точно.

С `expected.json` харнесс ставит **✅/❌** по каждому файлу и **краснит CI** при
расхождениях — тогда петлю «прогнал → починил → прогнал» можно гонять до зелёного,
а не вслепую.

## 3. Запусти
- **локально:** `python scripts/hybrid_eval.py tests/samples`
- **в CI:** Actions → «STT eval (сводка по записям)» → Run workflow

Без записей CI просто напишет «нет аудио» и не упадёт.
