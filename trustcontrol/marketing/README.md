# TrustControl — Лидогенерация + WhatsApp (рассылка и AI-продавец)

Полный конвейер: собрать лиды из 2ГИС → разослать первое сообщение в WhatsApp →
AI-бот сам отвечает на ответы и ведёт к созвону.

Все три скрипта используют **одну сессию WhatsApp** (папка `marketing/wa_session/`),
поэтому QR сканируется **один раз**.

## 1. Установка (один раз)

```bash
pip install -r marketing/requirements-marketing.txt
# либо: pip install selenium webdriver-manager openai requests
```
Нужен установленный **Google Chrome** (chromedriver selenium ставит сам).

## 2. Собрать лиды из 2ГИС

```bash
# из корня trustcontrol
python scripts\leadgen_2gis.py --niche coffee --analyze 50
```
Результат → `leads_<Город>_<ниша>.json` и `.csv` (название, телефон, рейтинг,
отзывы, «температура» лида, ссылка 2ГИС). Отсортировано по горячести.

Опции: `--city Астана`, `--niche pharmacy`, `--limit 200`.

## 3. Разослать первое сообщение в WhatsApp

```bash
# СНАЧАЛА dry-run — ничего не шлёт, только показывает кому и какой текст
python scripts\whatsapp_sender.py --leads leads_Алматы_coffee.json --top 20

# реальная отправка
python scripts\whatsapp_sender.py --leads leads_Алматы_coffee.json --top 20 --send
```

Первый запуск с `--send`: откроется WhatsApp Web → отсканируй QR с телефона
(Настройки → Связанные устройства). Сессия сохранится в `marketing/wa_session/` —
второй раз QR не нужен (и авто-ответчик подхватит ту же сессию).

## ⚠️ Защита от бана — ОБЯЗАТЕЛЬНО прочитай

WhatsApp банит за массовый холодный спам. Встроенные защиты рассыльщика:
- дневной лимит (`--limit`, по умолчанию 20)
- случайные паузы **90-240 сек** между сообщениями
- **каждое сообщение уникальное** (собирается из случайных кусков) — не один текст
- только рабочие часы (`--hours 10-19`)
- лог отправленных — повторно одному номеру не пишем
- заправки (LUKOIL, SOCAR и т.п.) отфильтровываются автоматически

Твои действия чтобы номер выжил:
- бери **ОТДЕЛЬНЫЙ** номер, не личный
- первые дни — по 20 сообщений, потом наращивай
- **отвечай на ответы быстро** (это и делает AI-продавец ниже)
- цель первого сообщения — не продать, а выйти на ЛПР и договориться о показе

## 4. AI-продавец (авто-ответы на ответы)

`whatsapp_autoresponder.py` крутится в фоне, читает входящие ответы от бизнесов,
сам отвечает через GPT как ассистент Данила и ведёт к одной цели — **забить созвон
с ЛПР**. Когда лид тёплый — пишет в `hot_leads.csv` и пингует тебя в Telegram.

```bash
cd marketing
set OPENAI_API_KEY=sk-...             # Windows (Linux: export OPENAI_API_KEY=...)
set TELEGRAM_BOT_TOKEN=...            # опц.: пинг о горячем лиде
set TELEGRAM_ADMIN_CHAT_ID=...        # опц.: твой chat_id
python whatsapp_autoresponder.py --poll 30
```

**Стоимость:** ~$0.0004 за сообщение (gpt-4o-mini). 500 диалогов ≈ $1.2.

**Что бот умеет:**
- отвечает на вопросы о продукте (по `PRODUCT_BRIEF` в коде — правь под себя)
- цену в переписке не называет, переводит на созвон («первый месяц бесплатно»)
- если пишет не владелец — выясняет как выйти на ЛПР
- ловит интерес → спрашивает удобное время → пишет в `hot_leads.csv` + Telegram

**⚠️ Риск:** Selenium-бот в WhatsApp — серая зона, селекторы WhatsApp иногда
меняются (если перестало читать чаты — чинить `_get_unread`/`_read_incoming`).
Отвечать тем, кто ответил сам — безопаснее холодной рассылки, но не злоупотребляй.
Для масштаба надёжнее официальный WhatsApp Business API (платный).

## Файлы

- `scripts/leadgen_2gis.py` — парсер 2ГИС → `leads_*.json` / `.csv`
- `scripts/whatsapp_sender.py` — рассылка первого сообщения по `leads_*.json`
- `marketing/whatsapp_autoresponder.py` — AI-продавец: сам отвечает на ответы
- `marketing/wa_session/` — общие данные (в .gitignore, не коммитятся):
  - `wa_profile/` — сессия WhatsApp (QR один раз, общая для обоих скриптов)
  - `wa_sent.log` — кому уже писали
  - `conversations.json` — история переписок бота
  - `hot_leads.csv` — горячие лиды с временем созвона

> `marketing/parse_2gis.py` — старый альтернативный парсер через официальный
> Catalog API 2ГИС (нужен ключ с dev.2gis.ru). Основной рабочий — `leadgen_2gis.py`.
