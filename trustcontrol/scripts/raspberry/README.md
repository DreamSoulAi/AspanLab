# TrustControl на Raspberry Pi — «коробка» для кассы

Выделенное устройство записи: не зависит от ноутбука клиента, не уходит в сон,
само перезапускается, копит записи офлайн и досылает при появлении сети.
Переиспользует тот же `monitor.py`, что и Windows-версия.

---

## Что купить (на одну кассу)

| Что | Зачем | Примерно |
|-----|-------|----------|
| Raspberry Pi Zero 2 W | мозг | 10–15к₸ |
| microSD 16–32 ГБ | система | 2–3к₸ |
| USB-микрофон (или USB-звуковая + микрофон) | звук | 3–8к₸ |
| Блок питания 5V/2.5A microUSB | питание | 2к₸ |
| Корпус (по желанию) | вид | 2к₸ |

> Pi Zero 2 W имеет только 512 МБ RAM — этого хватает (шумоподавление при
> нехватке памяти просто отключается). Если хочется запас — Pi 3A+/Pi 4.

---

## Шаг 1. Записать систему на карту (один раз)

1. Скачать **Raspberry Pi Imager**: https://www.raspberrypi.com/software/
2. ОС: **Raspberry Pi OS Lite (64-bit)** — без рабочего стола.
3. В настройках Imager (шестерёнка) сразу задать:
   - имя пользователя/пароль (чтобы зайти по SSH),
   - **включить SSH**,
   - можно вписать свой Wi-Fi (для первой настройки; рабочий Wi-Fi точки потом
     задаётся через `trustcontrol.txt`).
4. Записать карту, вставить в Pi, подключить USB-микрофон, включить.

## Шаг 2. Установить TrustControl (один раз на эту карту)

Зайти на Pi по SSH и выполнить:

```bash
sudo apt-get install -y git
git clone <URL_РЕПОЗИТОРИЯ> trustcontrol
cd trustcontrol/trustcontrol/scripts/raspberry
sudo bash install.sh
```

Скрипт сам поставит зависимости, автозапуск и положит шаблон `trustcontrol.txt`
на загрузочный раздел.

## Шаг 3. Размножить на все кассы (быстро)

Эта карта — **золотой образ**. Дальше не повторяете шаги 1–2:

1. `sudo poweroff`, вынуть карту.
2. На ПК снять образ карты (Win32DiskImager «Read» / `dd`) → один `.img`.
3. Записывать этот `.img` на новые карты — каждая уже готова.

---

## Шаг 4. Настроить под конкретную точку (на каждую коробку)

1. Вставить SD-карту в любой компьютер.
2. Открыть файл **`trustcontrol.txt`** (он на разделе, который виден сразу).
3. Вписать:
   ```
   api_key = ключ_этой_точки_из_личного_кабинета
   wifi_ssid = Wi-Fi_заведения        (пусто, если кабелем)
   wifi_password = пароль_Wi-Fi
   ```
4. Сохранить, вернуть карту в Pi, включить питание.

Через минуту записи пойдут на сервер. **Больше ничего настраивать не нужно.**

> Кабель Ethernet надёжнее Wi-Fi: воткнул — и `wifi_*` можно не заполнять.

---

## Установка микрофона (физически)

Микрофон — между кассиром и клиентом, 30–80 см, не вплотную к колонкам/кофемашине.
Цель — слышно ОБОИХ: и кассира, и клиента.

---

## Проверка и диагностика

```bash
# живой лог монитора (Ctrl+C чтобы выйти)
journalctl -u trustcontrol -f

# что применилось из trustcontrol.txt (Wi-Fi, ключ)
cat /var/log/trustcontrol-config.log

# список устройств записи (если микрофон не тот)
arecord -l

# перезапустить вручную
sudo systemctl restart trustcontrol
```

Накопленные при пропаже сети записи лежат в `/opt/trustcontrol/fails/` и
досылаются автоматически (фоновый ретрай каждые 5 минут).

## Обновление кода на коробке

```bash
cd ~/trustcontrol && git pull

```
